## DMIT 流量超量之后到底发生什么

这是搜 **DMIT 流量不够用** 最核心的问题，分系列说。

### LAX Pro 和 LAX EB 系列：降速，不停机

流量额度跑完之后，DMIT 不关你的服务器，而是把带宽限下来，无限量供你继续用。降速幅度根据套餐档位有区别：

- **入门档**（Pro.WEE / Pro.MALIBU / EB.INTRO / EB.WEE / EB.CORONA）：降至 2Mbps
- **TINY / Pocket 档**（Pro.TINY / Pro.Pocket / EB.TINY / EB.Pocket 等）：降至 4Mbps
- **MINI / MICRO 档**：降至 8Mbps
- **MEDIUM / LARGE / GIANT 档**：降至 10Mbps

4Mbps 够远程 SSH 进去管机器、跑后台、发工单；8Mbps 日常轻量任务几乎感觉不到；2Mbps 就只能说"够活着"。

买的如果是入门款，月底流量跑完那两天体验会差一些；MINI 及以上用下来，限速期间基本不影响正常维护操作。

如果你更习惯超量直接停机（比如你用脚本监控流量来手动续费），控制台里保留了切换选项，可以改回旧模式。

### T1 系列：限速但给保底带宽

T1 系列（洛杉矶、香港、东京都有）的超量策略略有不同——流量超了之后，限速到一个固定保底值继续无限跑：

- 香港和东京 T1.WEE：限速 50Mbps，无限流量
- 洛杉矶 T1.WEE：限速 100Mbps，无限流量

比 Pro/EB 系列的降速幅度好看得多。代价是 T1 系列本来就没有大陆优化线路，只走国际互联，国内访问晚高峰会有一定波动。如果你的业务主要面向海外，T1 超量后的体验反而比 Pro 系列强。

## DMIT 流量双向计费是坑点

很多人买了 Pro 或 EB 系列，发现流量消耗比预期快，原因在这：

**Pro 系列和 EB 系列双向计费**——进来的流量和出去的流量都算进去。你传一个 1GB 文件出去，同时收到 1GB 数据回来，计费里记的是 2GB，不是 1GB。

用来做代理/梯子的人尤其注意：客户端和服务器之间来来回回的流量会被翻倍计入，实际可用量比套餐标注的数字少很多。

**T1 系列单向取最大值**，也就是入站和出站取较大的那个数来计费，同样的使用场景下流量消耗比 Pro/EB 少。这是 T1 系列对做中转/落地节点的用户性价比更高的原因之一。

摘要一句话：**流量跑得比预期快，先检查是不是双向计费在吃你的额度。**

## 如何判断自己需要哪个档位

不同用途对流量的消耗差异很大，大概估一下：

**日常建站/博客/API 服务**：流量消耗相对可控，TINY 或 Pocket 通常够用。如果月均访问量在几万 PV 以下，加上双向计费的损耗，Pocket（1500-3000GB）一般撑得住。

**代理/梯子**：双向计费的影响在这个场景下最明显。建议流量额度至少留 1.5 倍的余量，单人使用 Pro/EB TINY（1000-1500GB）勉强够，多设备或多人共用直接考虑 Pocket 以上。

**视频传输/大文件同步**：这种场景月均流量动辄上 TB，STARTER（3000-5000GB）是起步档，稳当点上 MINI（5000-10000GB）。

**中转节点**：优先选 T1 系列，单向计费省流量，超量后还有 50-100Mbps 保底，做落地节点压力小得多。

问题是，多数人选套餐时会低估实际用量——买的时候觉得 1TB 应该够，跑起来发现根本不够。我自己一开始也这么估，吃过亏之后的经验是：预算允许就往上走一档，比月底紧张着撑强多了。

**Q：DMIT 流量不够用可以临时充值流量吗？**

**Q：LAX Pro 和 LAX EB 流量超量的限速规则一样吗？**
A：一样的，都是按套餐档位降速，TINY 降到 4Mbps，MINI/MICRO 降到 8Mbps，MEDIUM 以上降到 10Mbps。两个系列的区别在于线路——Pro 走三网 CN2 GIA，EB 走 CMIN2，同价位 EB 的流量额度比 Pro 多。

**Q：双向计费是怎么算的？会比预期消耗快多少？**
